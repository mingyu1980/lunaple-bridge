## 상세 지시

### 1. `functions/ai/compatibility.js`
```
- prokerala-auth.js 의 getProkeralaToken() 재사용 (신규 auth 로직 작성 금지)
- Prokerala API endpoint: GET https://api.prokerala.com/v2/astrology/kundli-matching
- 필수 파라미터:
  * girl_dob (ISO 8601, 예: 1995-06-15T10:30:00+05:30)
  * girl_coordinates (위도,경도 문자열, 예: "28.6139,77.2090")
  * boy_dob
  * boy_coordinates
  * ayanamsa (기본값: 1 = Lahiri)
- 응답에서 추출할 핵심 필드:
  * guna_milan.total_points (최대 36점)
  * guna_milan.percentage
  * guna_milan.is_match_making_compatible (boolean)
  * message (호환성 요약 텍스트)
- Firebase callable function으로 래핑 (onCall)
- 입력 유효성 검사: dob 형식 오류·좌표 누락 시 functions.https.HttpsError('invalid-argument') 반환
- 에러 핸들링:
  * Prokerala 4xx → 'invalid-argument'
  * Prokerala 5xx / 네트워크 오류 → 'unavailable'
  * 토큰 갱신 실패 → 'unauthenticated'
- 응답 로그: logger.info('[compatibility] points=%d compatible=%s', points, compatible)
```
